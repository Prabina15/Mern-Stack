
## Web
-WWW:World wide web
-System of interlinked docs acceded 
through the internet using browser.
-Website are made of HTML(Structure), CSS(Styling) and Javascript(Behaviour)

## Client Side Model
-Client:the browser or app that sends request in server.
-Server: The machine that process the requests and sends response(HTML, CSS, Data)


## HTML: Hypertext Markup Language
-1. Text elements

- Heading -> `h1` - `h6`
- Paragraph -> `p`
- Bold -> `strong`
- Emphasize -> `em`
- Sub-script -> `sub`
- Super-script -> `sup`
- Pre-formatted -> `pre`

2. Links and Media

- Anchor -> `a`
- Images -> `img`
- Videos -> `video`

3. Lists

- Ordered list -> `ol`
- Un-ordered list -> `ul`
- List item -> `li`

4. Misc

- Line break: `br`
- Divider: `hr`
- Iframe: `iframe`

5. Table

- `table`
- `tr`
- `td`
- `th`
- `thead`
- `tbody`
- `tfoot`

6. Form

- `form`
- `input`: text, number, email, tel, checkbox, radio, file, date, time, range, color, password, submit, search
- `label`
- `textarea`
- `select` & `option`


7. Inline & Block Elements
-Inline elements occupy only the required width. for e.g. a, img, video, span
-Block elements occupy covers full width of the screen. for e.g: h1-h6, p, div, li

8. Semantics
-All elements must be in lower case.
-Use proper elements as per their position and usage.
-Always add `alt` property in the img tag.
-Always add `title` property in the button tag especially with icon only button.
-HTML file names must be in kebab case.
-In each HTML file/page there must be atleast one h1 tag.
-Always format your code.

## CSS: Cascading Style Sheet

-Website styling like color, font-size, alignment, spacing

**Syntax**

```
<selector>{
    property:value;
    property:value;
    ....
}
```

**Selectors**
1. Element: h1, p, div |Lowest priority
2. Class: .class-name |Id> Class> Element
3. ID: #Id_name |Highest priority

**Class vs Id**
-For CSS, always use class
-for JS, use Id
